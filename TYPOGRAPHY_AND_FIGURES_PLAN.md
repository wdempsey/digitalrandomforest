# Typography & Figures Plan

Decisions made 2026-06-26, inspired by Maggie Appleton's site (Canela + fluid scale
+ hand-drawn illustration). We are matching the *feeling*, not cloning.

---

## Typography

**Body.** Switch the reading body from Inter (sans) to **Newsreader** — a warm,
literary serif designed for on-screen reading. This is the big shift toward the
Appleton feel: the site reads like a notebook, not a docs site. Spectral is the
calmer fallback; because fonts live in one place (`quartz.config.ts` →
`typography.body`), swapping is a one-line change.

**Headers.** Keep **Source Serif 4** for headings — it reads as a display serif and
pairs cleanly with Newsreader text. (Alternative if it feels like "two serifs
fighting": unify everything on Newsreader. Easy to try later.)

**Mono.** Keep IBM Plex Mono for code.

**Size & scale.** Maggie reads large and fluid. Move from the current 17px body to a
fluid scale:

- body: `clamp(18px, 1.05rem + 0.3vw, 20px)`, line-height ~1.62
- h1: `clamp(2.6rem, 2rem + 3vw, 3.6rem)`
- h2 / h3: scale down proportionally
- reading measure (`article` max-width): ~38rem — serifs read best slightly
  narrower than sans.

**Where it changes.** `typography.body` in `quartz.config.ts`; fluid `clamp()` rules
and the measure in the single `custom.scss` (canonical palette already there). Low
risk, reversible.

## Figures — hybrid engine

The figure pipeline (transformer + styles) already exists. It ingests both SVG and
raster from plain Markdown: `![alt](/assets/figures/x.svg "caption")`.

There is **no image-generation service connected**, and raster images can't be
rendered inside this workspace — so raster art is generated by *you* in an external
tool using a kit we prepare. SVG diagrams are authored directly in-repo.

### House style (shared by both engines)

- palette: sage `#6f8c7a`, charcoal `#2f2a26`, warm paper `#faf8f4`; muted, low
  contrast.
- feel: soft, loose, hand-drawn; generous negative space; quiet.
- captions: Inter, small, muted (already styled).
- always include alt text; SVGs get `<title>`/`<desc>`.

### Engine A — SVG hand-drawn diagrams (everyday backbone)

Authored in-repo as `content/assets/figures/*.svg`. Loose/irregular linework in the
house palette on transparent ground (paper shows through), dark-mode aware.
Best for conceptual figures. First targets, drawn from the anchor essay:

1. the compounding **ecosystem loop** (upgrade the existing sketch)
2. **force multiplier vs. force divider**
3. **frictionless reproducibility** — friction collapsing over time

### Engine B — painterly raster (occasional hero images)

A `FIGURE_STYLE_KIT.md` containing: one master style prompt, the palette hexes,
composition rules, aspect ratios, and negative prompts (no photorealism, no harsh
contrast, no text). Plus a per-essay **shot list**. You run the prompts in your image
tool (ChatGPT/DALL·E, Midjourney, Ideogram, …), save results as `.webp` to
`content/assets/figures/`, and reference them in Markdown. Consistency tip: reuse the
master prompt and fix a seed / reference image across an essay.
