# Quartz Design Target

This document describes the visual and structural target for the Quartz digital garden.

Codex should read this before generating layout or styles.

---

# Design Inspiration

The design is inspired by the spirit of Maggie Appleton’s digital garden.

However, this site is not a clone.

The goal is a similar feeling adapted for:

- mathematics
- statistics
- code
- academic writing

---

# Target Mood

The site should feel like:

- a quiet study
- a research notebook
- a reflective intellectual space
- a long-term thinking environment

The visual tone should be:

- warm
- calm
- minimal
- editorial

Avoid anything that feels like:

- developer documentation
- dashboards
- SaaS UI
- technical blogs with heavy styling

---

# Typography

Preferred structure:

Headers: serif  
Body text: clean sans-serif

Examples:

Serif options:
- Lora
- Fraunces
- Crimson

Sans-serif options:
- Inter
- Source Sans
- System UI

Headers should feel slightly literary.

Body text should prioritize readability.

---

# Colors

Use a restrained palette.

Suggested direction:

Background  
soft parchment or warm off-white

Examples:
#fdfaf6
#f8f5ef

Text color  
dark charcoal rather than pure black

Examples:
#2c2c2c

Accent colors  
very subtle muted tones.

Avoid bright or saturated colors.

---

# Layout

Reading experience is the most important priority.

Key rules:

- comfortable line length
- generous margins
- clear vertical rhythm
- strong typographic hierarchy

Article text should not be too wide.

Whitespace should create calm rather than emptiness.

---

# Navigation

Navigation should be simple.

Primary navigation should emphasize:

Garden  
Essays  
Notes  
Patterns  
Library  
About  
Now

Avoid overly complex navigation systems.

---

# Metadata

Metadata should be visible but subtle.

Examples:

Published  
Updated  
Status  
Version

Metadata should not dominate the page.

---

# Graph View

Graph view is optional exploration.

It should exist but remain visually secondary.

The main experience should always prioritize reading.

---

# Code Blocks

Code blocks should be styled cleanly.

Avoid overly dark developer-style themes.

They should integrate visually with the page.

---

# Math Rendering

Math should render clearly and elegantly.

Block equations should have comfortable spacing.

Inline math should not disrupt line height.

---

# Sketches

Some essays may include one minimal sketch.

Sketches should be:

- small
- quiet
- minimal
- supportive of the text

They should not dominate the page.

---

# Homepage

The homepage should feel editorial.

Possible elements:

- short introduction
- recent essays
- selected notes
- featured ideas

Avoid auto-generated dashboards.

The homepage should feel curated.
